# Тестовое задание QA Engineer

Решение тестового задания. Каждый блок лежит в своей директории с файлом `readme.md`, где описан ход решения. Везде, где применялся ИИ, рядом есть моё ревью, а не просто скопированный ответ модели.

## Структура

| Блок | Директория | Что внутри |
|---|---|---|
| 1. Тестирование требований (ЭТрН) | [block1_etrn](block1_etrn/readme.md) | Чек-лист и тест-кейсы по BPMN-процессу, проверки формы, негативные сценарии, баг-репорты |
| 2. Проектирование тестов (AI Prompting) | [block2_ai_prompting](block2_ai_prompting/readme.md) | Промпт, сырой ответ ИИ, моё ревью, плюс [граничные условия](block2_ai_prompting/boundary_conditions.md) |
| 3. Анализ данных (SQL) | [block3_sql](block3_sql/readme.md) | Запросы на PostgreSQL с разбором подводных камней |
| 4. Тестирование API | [block4_api](block4_api/readme.md) | Чек-лист для контроллера /api/v2/orders по контракту и гайдам Google / Microsoft |

## Какие блоки выбраны

Обязательная часть это блоки 1 и 2. Из блоков на выбор задание просило взять любые два, я выбрал блоки 3 и 4.

## Вопросы по бизнес-логике

Места, где для точного ответа нужны исходные артефакты (XSD-схема формата ЭТрН, отдельные детали контракта API), помечены прямо в тексте тегом `[свериться ...]`. На реальном проекте это были бы Issue в репозитории.
